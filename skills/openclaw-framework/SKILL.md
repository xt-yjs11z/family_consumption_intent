# OpenClaw AI Assistant Framework

> 基于 Work-Fisher 的 OpenClaw AI 助手框架 v3.0

## 概述

这是一个完整的 AI 助手系统，包含 7 步深度使用法、4 层模型池、3 层记忆体系。

## 核心能力

### 1. 🚀 OpenClaw 7步深度使用法

#### 步骤1: 智能备份
- 定期备份核心配置
- 7天轮换备份策略

#### 步骤2: 四层模型池
根据任务选择模型：
- **高速池**: 日常对话、快速响应
- **智能池**: 复杂推理、编程
- **文本池**: 长文本写作
- **视觉池**: 图像/视频处理

#### 步骤3: 会话管理
- 相关度高 → 保持会话
- 相关度低 → 开启新会话

#### 步骤4: 上下文压缩
- 删除重复内容
- 简化礼貌用语
- 转换为结构化格式
- 节省约 22% tokens

#### 步骤5: 任务铁律 ⚡
**遇到问题时**:
1. 直接执行
2. 换个方法
3. 查阅文档
4. 搜索方案
5. 组合方法
→ **至少尝试5轮再求助用户**

**安全边界** (必须先问):
- 删除操作
- 系统配置
- 外部发送
- 支付/授权

#### 步骤6: 陌生任务
学习优先级:
1. ClawHub (50%)
2. GitHub (30%)
3. YouTube/B站 (15%)
4. 其他 (5%)

#### 步骤7: 自我进化
- 每日生成进化报告
- 提取关键决策到 MEMORY.md
- 固化重复使用的解决方案

---

### 2. 🧠 三层记忆体系

| 层级 | 位置 | 用途 |
|:---|:---|:---|
| L1 | 工作记忆 | 当前会话临时信息 |
| L2 | memory/YYYY-MM-DD.md | 每日事件和决策 |
| L3 | MEMORY.md | 长期核心经验 |

---

### 3. 💓 Heartbeat 维护

**每30分钟**:
- 检查紧急事项
- 整理记忆
- 清理日志

**每日**:
- 检查昨日memory文件
- 提取重要决策

**每周** (周日):
- 回顾MEMORY.md
- 清理30天前记忆

## 使用指南

### 任务铁律执行

遇到问题时，按顺序尝试：
1. 直接执行当前方案
2. 换个方法/思路
3. 查阅 OpenClaw 文档
4. 搜索解决方案
5. 组合多种方法

只有超过 5 轮尝试或遇到安全边界操作时才询问用户。

### 模型选择

根据任务类型选择模型池：

| 任务 | 推荐模型池 |
|:---|:---|
| 日常问答 | 高速池 |
| 代码/推理 | 智能池 |
| 写作/长文 | 文本池 |
| 看图/图像 | 视觉池 |

### 陌生任务处理

1. 先尝试用已有能力解决
2. 如果无法解决，搜索 ClawHub
3. 安装相关 skill
4. 如果没有，创建自定义 skill
5. 固化解决方案

## 定时任务 (可选配置)

```bash
# Heartbeat 检查 (每30分钟)
*/30 * * * * openclaw heartbeat

# 每日进化报告 (22:00)
0 22 * * * openclaw agent --message "生成进化报告"

# 模型健康检查 (每6小时)
0 */6 * * * openclaw agent --message "检查模型状态"
```
