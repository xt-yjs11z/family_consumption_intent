# 画像实时更新处理器

## 功能

自动识别对话中的家庭画像变更信息，并实时更新。

## 触发条件

用户在对话中提及任何变更信息时触发。

## 变更类型与识别

### 1. 手机变更
**关键词**: 换手机、买手机、手机换成、买了部手机
**提取**: 品牌+型号+使用者
**示例**: "我手机换成iPhone 15了" → 更新 members[phone]

### 2. 电子设备变更
**关键词**: 买了台电脑、换电脑、新电脑
**提取**: 设备类型+品牌+型号
**示例**: "买了台MacBook Pro" → 更新 appliances.电子产品

### 3. 家电变更
**关键词**: 换冰箱、买空调、冰箱换成
**提取**: 家电类型+品牌+型号
**示例**: "冰箱换西门子了" → 更新 appliances.大家电

### 4. 家庭成员变更
**关键词**: 添了、生了、二胎、家里来人了
**提取**: 新成员信息
**示例**: "家里添二胎了" → 更新 members + stage

### 5. 职业/身份变更
**关键词**: 上高中、上大学、工作了、退休了
**提取**: 成员+新身份
**示例**: "孩子上高中了" → 更新 members[].job

### 6. 收入变更
**关键词**: 涨工资、加薪、收入高了、工资涨了
**提取**: 收入变化
**示例**: "最近涨工资了" → 更新 income_level

### 7. 消费风格变更
**关键词**: 要省钱、不乱买了、开始节俭
**提取**: 新的消费风格
**示例**: "现在开始省钱" → 更新 consume_style

### 8. 健康变更
**关键词**: 体检、出问题了、查出、有病
**提取**: 成员+健康问题
**示例**: "我爸体检出血糖高" → 更新 health

### 9. 资产变更
**关键词**: 买车、买房、换车
**提取**: 资产类型
**示例**: "买了辆车" → 更新 assets

### 10. 住址变更
**关键词**: 搬家、搬新家、搬到
**提取**: 新地址相关信息
**示例**: "搬新家了" → 提示更新家具家电

---

## 更新流程

```
1. 监听用户消息
2. 检测变更关键词
3. 提取变更信息
4. 识别涉及成员
5. 更新画像
6. 记录变更历史
7. 确认更新
```

---

## 更新确认模板

| 变更类型 | 确认话术 |
|----------|----------|
| 手机 | "已记录：你把手机从[旧]换成[新]了" |
| 设备 | "已更新：家里新增了[设备]" |
| 家电 | "已更新：[家电]换成[品牌]了" |
| 成员 | "已记录：家里新增了[成员信息]" |
| 职业 | "已更新：[成员]现在是[职业]" |
| 收入 | "已记录：家庭收入变为[等级]" |
| 消费风格 | "已更新：消费风格改为[风格]" |
| 健康 | "已记录：[成员][健康状况变化]" |
| 资产 | "已记录：新增资产[资产]" |

## 变更历史记录格式

```json
{
  "family_id": "family_001",
  "updates": [
    {
      "timestamp": "2026-03-18T10:30:00Z",
      "type": "phone",
      "member_id": "member_003",
      "field": "phone",
      "old_value": "iPhone 13",
      "new_value": "iPhone 15",
      "source": "对话",
      "confirmed": true
    }
  ]
}
```

---

## 注意事项

1. **模糊匹配**: 如信息不完整，追问确认
2. **隐私保护**: 敏感信息使用抽象标签
3. **用户确认**: 更新后必须告知用户
4. **历史追溯**: 所有变更都记录在案
