# 家庭消费意图识别技能

## 技能名称

family-intent

## 功能描述

识别家庭成员的消费意图，补全缺失信息，提供个性化选购建议。

## 触发条件

用户发送消费相关需求，如：
- "给孩子买手机"
- "买空调"
- "过年给父母买礼物"
- "想买点什么"

## 处理流程

### 1. 意图识别

解析用户消息，提取：
- **target**：消费对象（自己、孩子、父母、家庭）
- **object**：商品（手机、电脑、空调等）

### 2. 槽位检查

检查必填字段是否完整：
- target（消费对象）
- object（商品）

### 3. 决策

| 状态 | 动作 |
|------|------|
| 信息完整 | 输出识别结果 + 预算预测 + 选购建议 |
| 信息不完整 | 只追问，不显示其他内容 |

### 4. 追问规则

⚠️ **追问时只显示问题，不显示选项列表！**

```
用户：我想买礼物
Agent：想买什么商品？
```

追问优先级：
1. 消费对象（target）
2. 商品（object）

## 必填字段

| 字段 | 说明 | 示例 |
|------|------|------|
| target | 消费对象 | 自己、孩子、父母、家庭 |
| object | 商品 | 手机、电脑、空调 |

## 可选字段

| 字段 | 说明 | 来源 |
|------|------|------|
| intent_type | 商品类别 | 自动识别 |
| scene | 使用场景 | 自动预测 |
| budget | 预算范围 | 根据家庭画像预测 |

## 模糊词处理

以下词汇不识别为商品，触发追问：
- 礼物、东西、物品、物件
- 买点、看看、推荐

## 场景词库

**节日**：春节、元宵、清明、五一、端午、中秋、国庆、情人节、七夕、520、母亲节、父亲节、六一、生日、元旦、圣诞节

**坏了**：坏了、破了、小了、不能用、用完了

**换季**：春天、夏天、秋天、冬天、天冷了、天热了

**促销**：双11、618、打折、促销、优惠

## 消费对象优先级

孩子 > 父母 > 家庭 > 自己

## 预算预测规则

| 消费风格 | 数码 | 家电 | 教育 | 服饰 |
|----------|------|------|------|------|
| 节俭 | 1000-2000 | 2000-3000 | 50-100 | 100-300 |
| 理性 | 2000-4000 | 3000-5000 | 100-300 | 300-500 |
| 适度 | 3000-5000 | 4000-6000 | 200-500 | 500-1000 |
| 大方 | 5000-8000 | 6000-10000 | 500-1000 | 1000-2000 |
| 奢侈 | 10000+ | 10000+ | 1000-3000 | 2000-5000 |

## 用户加载规则

### 1. 根据 userId 加载配置

每次请求携带 userId 参数：
- 加载 USER/USER_{userId}.md
- 加载 memory/{userId}.md

### 2. 回退规则

- 用户配置不存在 → 使用默认 USER.md
- 记忆文件不存在 → 创建新的

## 推荐原则

- 禁止推荐具体产品型号
- 只提供选购方向和建议
- 结合家庭消费风格给出合理建议

## 示例

### 完整对话

```
用户：给孩子买手机
Agent：✅ 识别完成！
  - 消费对象：孩子
  - 商品：手机
  - 类型：digital
  - 预算：5000-8000
  - 建议：...
```

### 需要追问

```
用户：我想买礼物
Agent：想买什么商品？
```

- 场景：日常

- 时间：计划购买
